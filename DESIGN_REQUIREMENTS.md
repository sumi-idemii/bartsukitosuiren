# デザイン要件（Bar 月と睡蓮）

このドキュメントは、本プロジェクトの UI/UX の意図と実装基準を共有するためのものです。スタイルの一次ソースは `src/index.css` の CSS 変数とクラスです。

---

## 1. 目的とブランドのトーン

- **対象:** ウイスキーバー「月と睡蓮」の公式 Web サイト。
- **ビジュアル方針:** **インダストリアル・ノワール（工業的暗黒美）** — スチールやコンクリート、煤と錆を連想する**無機質な暗さ**と、倉庫や埠頭の**一点の灯り**のようなノワール的ライティング。派手なプロモ色より、**冷たい灰青**と**わずかな琥珀（ウイスキー／錆）アクセント**で締める。
- **言語:** 日本語を主とし、ナビゲーションラベル等では英字大文字（例: `DRINK`）を併用してもよい。
- **一貫性:** 新規画面・コンポーネントは既存のタイポグラフィ階層（`.eyebrow` / `.h1` / `.h2` / `.bodyText`）とカラートークンに従うこと。

---

## 2. カラーシステム

| 役割 | 変数 | 値の意図 |
|------|------|----------|
| 背景 | `--bg`, `--bg-deep` | 純黒ではなく**鋼鉄の暗箱**（例: `#0a0b0e` 系）。対角・放射グラデで深度を付ける。 |
| 前景 | `--fg` | わずかに冷たいオフホワイト（`#e8eaef` 系）。 |
| 補助テキスト | `--muted` | 灰青がかった説明文・アイブロー。 |
| 区切り線 | `--hairline`, `--hairline-faint` | **ヘアラインではなく**スチール継ぎ目のような淡い灰青境界。 |
| フォーカス | `--focus` | キーボードフォーカス時の輪郭。 |
| アクセント | `--accent`, `--accent-dim`, `--accent-glow` | **琥珀／錆**。ナビ現在地・ヒーローリンク下線・ヘッダー影など**差し色のみ**に使う。 |

**禁止に近いこと:** 彩度の高いアクセントで画面全体を染めること、背景を明るい単色に切り替えること。

---

## 3. タイポグラフィ

| 用途 | 変数 / クラス | 方針 |
|------|----------------|------|
| UI・本文 | `--sans` | システム UI + 日本語ゴシック系。16px 基準。 |
| 和文見出し・店名 | `--serif-tight` | `'Zen Old Mincho'`。格式とバーの品格。 |
| 英字ラベル・ナビ・ヒーローメニュー・アイブロー | `--display` | **`Oswald`** を優先（Google Fonts）。狭字・大文字で工業的な印字感。 |
| フッター細字 | `.fineprint` | `--display` で字間広め。 |

**要件:** 行長は長文でおおよそ **72ch 以内**（`.sectionBody` に準拠）。折り返しは `overflow-wrap` を考慮し、モバイルでは URL 等がレイアウトを壊さないこと。

---

## 4. レイアウトとスペーシング

| 項目 | 値 |
|------|-----|
| コンテナ最大幅 | `--container`: **1040px** |
| 横パディング | `--pad-x`: **24px**（狭い画面では CSS で変調あり） |
| 縦リズム | メイン `.main` は上下 **40px / 64px**。セクション間は `.stack` の **42px** 等で統一。 |

- **トップページ:** ファーストビューは **100svh** 程度。背景ロゴは装飾として固定配置し、スクロールで不透明度が変化しても可（実装に準拠）。
- **下層ページ:** ヘッダーは **sticky**、本文は `.container` 内に収める。

---

## 5. 視覚効果（テクスチャ・質感）

- **粒子:** `body::before` のフラクタルノイズは**粗め**（コンクリ床上の塵・煤）を想定。`soft-light` で重ね、本文の可読性を損なわないこと。
- **ヴィネット:** `body::after` で**周辺を沈め**、中央に視線を集めるノワール処理。
- **パネル:** ヘッダー・ヒーローメニューは**薄い内側ハイライト**と**深いドロップシャドウ**で厚みのある金属板を連想させる。
- **ChippedText:** テクスチャは `--paint-peel`（`/textures/menu-mask.png`）。フィルタで**わずかに sepia** をかけ、酸化膜／すり傷のような読みに寄せる。
- **アクセシビリティ:** 画面読み上げ用に `sr-only` を維持すること。

---

## 6. ナビゲーションとインタラクション

- **ホーム:** ヒーロー下部のメニューは半透明パネル + ブラー。リンクホバーは背景のわずかな明るさと下線アニメーション。
- **下層:** `.navLink` の現在地は `.isActive` でボーダーと薄い背景を付与。
- **フォーカス:** `a:focus-visible`, `button:focus-visible` は **2px アウトライン + 3px オフセット**（`index.css` に準拠）。
- **スキップリンク:** 「本文へスキップ」をキーボード利用時に表示可能に維持する。

---

## 7. コンテンツブロック

- **セクション:** `.section` は上ボーダー + うすい**内側ハイライト**で区切る。
- **メニュー一覧:** `.menuRow` は **2 カラム**、モバイルでは **1 カラム**。
- **定義リスト:** `.dlRow` の `dt` は英字ラベル向けに `--display`。
- **ギャラリー:** 角丸は**控えめ（約 6px）**。枠は `--hairline`、画像は彩度を少し落として**地下展示風**に。

---

## 8. アクセシビリティ

- コントラスト: 暗背景上での `--fg` / `--muted` は、意味のある文言について **読みにくさがないこと** を確認する。
- 装飾画像は `alt=""` と `aria-hidden` を適切に使う。
- インタラクティブ要素は **フォーカス可能**であり、論理的な **見出し階層** を保つ。

---

## 9. レスポンシブ

- **ブレークポイントの目安:** 900px（ナビ縦積み・ギャラリー 2 カラム）、600px（ヒーローメニュー・メニュー行・dl の 1 カラム化）。
- 横スクロールは `body` の `overflow-x: hidden` に頼らず、コンテンツ側で **はみ出し原因を解消**すること。

---

## 10. 変更時の注意

- 新しい色やフォントを追加する場合は **` :root` に変数を追加**し、コンポーネントからは変数参照に寄せる。
- ChippedText のテクスチャ差し替えは **アセットと変数の両方**をドキュメントまたはコミットメッセージで追えるようにする。
- 英字用に **Oswald** を追加しているため、`index.html` の Google Fonts リンクと `--display` のスタックを整合させること。
